2017.8.19离线赛题解

LOVE

看不懂。。第一题为什么要放这么简单

dp的定义和转移太显然了。

主要的坑点还是在高精上，题目太简单反而很多人忽视了高精。我是写第二题的时候，回来发现$90$分提示不超过$long long$才意识。

比赛有$4$个人没有写高精，应该都是大意了，跑去第二三题。

太简单也不一定简单，考的就是细心。哪怕只是手造数据也还是能排查掉很多问题(见代码处)。

毕加猪

将题目转化一下，就是求有负边的有向图中，从$s$到$e$或者$e$到$s$是否存在一条距离小于等于$-T$的路径。

这样，如果路径上存在负环，就可以不断刷，使得距离无穷小。否则就只能找两点的最短距离。

这题的难点就在如何判断一个点是否在负环上，不会直接$gg$，事实上，除了$dijkstra$的求最短路都是可以用来求解含负边的最短路问题。

$Bellman-ford$和$SPFA$可以用于判断路径上是否有负环，所以先求出$s$到$e$的路径上的点，在这些点上找负环。

$Floyd$的话，当$dis[i][i]<0$时说明点$i$经过负环，只要判断点$i$是否能出现在路径中即可。

#include<stdio.h>
#include<iostream>
#include<algorithm>
#include<string.h>
#include<vector>
#define M 205
#define oo 1000000000
#define LL long long
using namespace std;
int n,m,T,s,e;
LL dis[M][M];
int main(){
//  freopen("pig.in","r",stdin);
//  freopen("pig.out","w",stdout);
    int cas;
    scanf("%d",&cas);
    while(cas--){
        scanf("%d%d%d",&n,&m,&T);
        for(int i=1;i<=n;i++){
            for(int k=1;k<=n;k++)dis[i][k]=1ll*oo*oo;
        }
        for(int i=1;i<=m;i++){
            int x,y,z;
            scanf("%d%d%d",&x,&y,&z);
            dis[x][y]=min(dis[x][y],1ll*z);
        }
        scanf("%d%d",&s,&e);
        for(int i=1;i<=n;i++){
            for(int k=1;k<=n;k++){
                if(dis[k][i]==1ll*oo*oo)continue;
                for(int j=1;j<=n;j++){
                    if(dis[i][j]==1ll*oo*oo)continue;
                    dis[k][j]=min(dis[k][j],dis[k][i]+dis[i][j]);
                }
            }
        }
        bool ok=0;
        if(dis[s][e]<=-T)ok=1;
        if(dis[e][s]<=-T)ok=1;
        if(!ok){
            for(int i=1;i<=n&&!ok;i++){
                if(dis[i][i]<0){
                    if(dis[s][i]!=1ll*oo*oo&&dis[i][e]!=1ll*oo*oo)ok=1;
                    if(dis[e][i]!=1ll*oo*oo&&dis[i][s]!=1ll*oo*oo)ok=1;
                }
            }
        }
        if(ok)puts("yes");
        else puts("no");
    }
    return 0;
}

k倍数字

第二类数据挂掉了，得到的教训就是模和除的速度是非常慢的，每次$4$位和每次$6$位的速度竟然相差$2s$。

考虑$[1,K*R]$内的数字$i$，当$i \ mod \ K=0$ 且$f(i)=f(i/K)$时成立，所以定义$dp[i][k][j][t]$为当前正在考虑第$i$位，当前数字模$K$为$k$，当前数字与除$K$的$f$的差为$j$，是否已经比上界要小。

转移方程的难点还是在$j$上。用了高精除法的思想，对于当前枚举的数字$d$，当前数字除$K$的数值可以表示$(k*10+d)/K$。
